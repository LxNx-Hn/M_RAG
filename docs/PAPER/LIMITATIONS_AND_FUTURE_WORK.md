# M-RAG 한계와 후속 과제

- 기준일 2026-04-26
- 목적 논문 본문과 발표 질의응답에서 현재 한계를 일관되게 설명하기 위한 기준 문서

## 현재 한계

### 검색과 데이터 범위

- 인용 추적은 접근 가능한 메타데이터와 외부 소스 범위에 영향받음
- 문서 업로드 기반 흐름이 중심이라 웹 전체 검색 시스템으로 주장하기 어려움
- 실험용 데이터셋과 업로드 논문 구성에 따라 결과가 달라질 수 있음

### 생성과 추론 자원

- Base 모델 사용은 더 큰 GPU 자원에 의존
- 로컬 12GB급 GPU에서는 Mini 기준 운영이 안정적
- 생성 품질은 검색 결과와 디코딩 설정에 민감

### 운영과 서비스화

- 공개 서비스 보안 완성은 논문의 핵심 기여 범위 밖
- 대규모 다중 사용자 운영 검증은 별도 트랙
- 브라우저와 배포 인프라 수준 품질은 연구 프로토타입과 운영 제품의 중간 단계

## 논문에서 권장하는 한계 서술 방식

- 구현 범위 밖 항목은 숨기지 않고 분리해서 적기
- 현재 저장소에서 검증된 범위만 본문 핵심 주장으로 사용
- 실험 미완료 항목은 본문 결론이 아니라 향후 과제로 이동

## 후속 과제

- 문서 유형별 라우팅 성능 정량 비교 강화
- BM25와 벡터 검색의 컬렉션별 영속화 고도화
- 더 넓은 학술 도메인과 한국어 데이터셋 확장
- 인용 추적의 소스 범위 확장
- 서비스 운영 보안과 관측성 강화
