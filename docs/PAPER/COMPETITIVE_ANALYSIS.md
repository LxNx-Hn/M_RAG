# M-RAG 비교 포지셔닝

- 기준일 2026-04-26
- 목적 논문 관련 문서에서 M-RAG의 위치를 과장 없이 설명하기 위한 비교 프레임
- 주의 외부 시스템 세부 기능은 시점에 따라 달라질 수 있으므로 범주 중심으로 서술

## 비교 축

- 질의 라우팅이 있는가
- 학술 문서 구조를 반영하는가
- 검색과 생성 제어를 함께 다루는가
- 한국어 중심 실험 경로가 있는가
- 로컬 재현 실행 경로가 문서화되어 있는가

## 범주별 위치

| 범주 | 예시 | 강점 | M-RAG와의 차이 |
|---|---|---|---|
| 범용 문서 QA 도구 | 범용 노트형 문서 QA | 빠른 사용성 | 질문 유형별 파이프라인과 생성 제어 노출이 약함 |
| 논문 검색 중심 QA 도구 | 논문 탐색형 RAG 도구 | 문헌 탐색과 인용 보조 | CAD SCD 같은 디코딩 제어와 로컬 실험 경로가 핵심은 아님 |
| 검색 연구 계열 RAG | 하이브리드 검색 중심 연구 | 검색 품질 개선 | 라우팅과 생성 제어까지 일체형으로 묶이지 않는 경우가 많음 |
| 디코딩 연구 계열 | Contrastive Decoding 계열 | 생성 안정화 | 전체 학술 QA 시스템과 실험 자동화까지 포함하지 않음 |
| 다국어 임베딩 계열 | 다국어 검색 모델 계열 | 검색 기반 모델 품질 | 문서 QA 파이프라인 전체 기여와는 범위가 다름 |

## M-RAG를 설명하는 한 줄

- M-RAG는 학술 문서 QA를 위해 질의 라우팅, 하이브리드 검색, 생성 제어, 로컬 실험 자동화를 한 저장소 안에 묶은 구현 중심 시스템

## 논문에서 강조할 차별점

- 질문 유형별 파이프라인 분기
- 검색 단계와 생성 단계의 기여를 분리해 설명 가능
- 한국어 질의와 문서 유형 확장 흐름 포함
- 실험 실행과 표 생성 경로가 저장소 안에 명시

## 논문에서 과장하면 안 되는 지점

- 특정 외부 서비스보다 무조건 우월하다는 표현
- 웹 전반 검색 도구와 동일 범주의 비교
- 인용 추적이 모든 출판사와 모든 PDF를 포괄한다는 표현
