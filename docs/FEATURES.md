# M-RAG 기능 현황 & NotebookLM 비교

> 논문톤 아닌 실제 기능 기준. 뭐가 되고, 뭐가 안 되고, 뭘 더 할 수 있는지.

---

## 지금 M-RAG가 할 수 있는 것

### 문서 업로드 & 관리

| 기능 | 설명 |
|------|------|
| PDF 업로드 | 드래그앤드롭으로 PDF 올리면 자동 파싱 + 인덱싱 |
| DOCX/TXT 업로드 | Word 파일, 텍스트 파일도 처리 |
| 다중 문서 | 여러 논문/문서 동시 업로드, 목록에서 관리 |
| 문서 삭제 | 업로드한 문서 개별 삭제 |
| 섹션 태그 표시 | 업로드된 논문의 인식된 섹션(Abstract, Method 등)을 카드에 표시 |

### PDF 뷰어

| 기능 | 설명 |
|------|------|
| PDF 렌더링 | 오른쪽 패널에 PDF 원문 표시 |
| 출처 하이라이트 | 답변에서 인용한 부분을 PDF에 노란색 하이라이트 |
| 출처 클릭 이동 | 답변 하단 출처 클릭 → PDF 해당 페이지로 이동 |
| 참고문헌 탭 | PDF/참고문헌 탭 전환. 인용된 논문 목록 표시 + 다운로드 버튼 |

### 질의응답 (6개 파이프라인 자동 분기)

| 질문 유형 | 예시 | 어떻게 동작하는지 |
|-----------|------|------------------|
| 단순 질문 (A) | "이 논문의 데이터셋이 뭐야?" | HyDE로 질문 확장 → 벡터+키워드 하이브리드 검색 → 답변 |
| 섹션 질문 (B) | "실험 결과가 어떻게 나왔어?" | 해당 섹션(result)만 필터링해서 검색 |
| 비교 질문 (C) | "두 논문 차이가 뭐야?" | 논문별 병렬 검색 → 비교 표 형태로 답변 |
| 인용 추적 (D) | "참고문헌 분석해줘" | arXiv에서 인용 논문 자동 다운로드 + 인덱싱 |
| 전체 요약 (E) | "이 논문 전체 요약해줘" | 섹션별 5회 검색 → 구조화된 요약 |
| 퀴즈 생성 (F) | "이 내용으로 문제 만들어줘" | 객관식 5문제 + 정답 + 해설 자동 생성 |

### 라우트 배지 (투명성)

- 매 답변 상단에 "Route A: 단순 QA", "Route D: 인용 추적" 같은 배지가 표시됨
- 어떤 파이프라인으로 답변이 생성됐는지 사용자가 볼 수 있음
- NotebookLM은 이런 게 없음 (블랙박스)

### 추천 질문 말풍선

- 매 답변 아래에 후속 질문 3개가 pill 버튼으로 표시됨
- 클릭하면 바로 그 질문이 전송됨
- 파이프라인별로 다른 추천 (예: 퀴즈 답변 후에는 "정답 해설", "더 어려운 문제" 제안)

### 환각 억제 (CAD)

- 답변 생성 시 LLM이 "아는 척" 하는 걸 자동으로 억제
- α 값(0~1) 조절 가능 — 높을수록 검색된 내용에만 충실
- NotebookLM은 이 제어가 불가능

### 언어 이탈 방지 (SCD)

- 영어 논문을 읽고 한국어로 답변할 때, 갑자기 영어로 빠지는 걸 방지
- β 값으로 강도 조절

### 문서 구조 보존

| 요소 | 처리 방식 |
|------|----------|
| 코드 블록 | monospace 폰트 감지 → 코드 블록으로 묶어서 보존 |
| 표 | markdown 표로 변환 + pymupdf fallback |
| 수식 | 수학 폰트/유니코드 기반 감지 → 분할 금지 |
| BNF/EBNF 문법 | 코드 블록으로 인식되어 원형 보존 |

### 문서 유형 자동 판별 (4종)

| 유형 | 감지 기준 | 활성화되는 기능 |
|------|----------|---------------|
| 논문 (paper) | Abstract, Method, Conclusion 등 | 섹션 필터 검색, arXiv 인용 추적 |
| 강의/교재 (lecture) | Definition, Theorem, Example, Chapter 등 | 정의/정리/증명 블록 보존 |
| 특허 (patent) | 청구항, 배경기술, 발명의 상세한 설명 등 | 인용 특허 파싱, Google Patents 연동 |
| 일반 문서 (general) | 위에 해당 안 되면 | 고정 크기 청킹 + 단순 QA |

### 특허 지원

| 기능 | 설명 |
|------|------|
| 특허 섹션 인식 | 청구항, 배경기술, 상세한 설명 등 11종 자동 감지 |
| 인용 특허 파싱 | KR/US/JP/EP/WO 특허 번호 자동 추출 |
| Google Patents 연동 | 인용 특허 메타데이터 자동 수집 |
| 유사 특허 검색 | 청구항 키워드 기반 유사 특허 후보 제시 |

### 기타

| 기능 | 설명 |
|------|------|
| 회원가입/로그인 | JWT 기반 인증 |
| 대화 기록 | 과거 대화 목록 보기, 이어서 대화 |
| 한/영 UI | i18n 다국어 지원 |
| SSE 스트리밍 | 답변이 실시간으로 글자씩 출력 |
| 마크다운 렌더링 | 표, 코드블록, 볼드 등 마크다운 답변 |
| Docker 배포 | `docker compose up` 한 줄로 전체 시스템 실행 |

## NotebookLM이 되고 M-RAG가 안 되는 것

| 기능 | 난이도 | 구현 가능? |
|------|--------|-----------|
| 오디오 오버뷰 (팟캐스트) | 높음 | 가능하지만 TTS 모델 필요 (비용 큼) |
| YouTube 링크 입력 | 중간 | youtube-transcript-api로 자막 추출 → 텍스트로 처리 가능 |
| 웹 URL 입력 | 낮음 | BeautifulSoup 크롤링 → 텍스트 추출 가능 |
| Google Drive 연동 | 중간 | OAuth + Drive API, 근데 굳이? |
| PPT 자동 생성 | 중간 | python-pptx + LLM 프롬프트 |
| 플래시카드 | 낮음 | Pipeline F 변형 (앞뒤 카드 형태 출력) |
| 소스 50개 제한 | - | M-RAG도 제한 없으나 실용적으로 ~30편 |

## 현실적으로 추가할 수 있는 기능 (난이도순)

### 쉬움 (1~2시간)

| 기능 | 어떻게 | 효과 |
|------|--------|------|
| **웹 URL 입력** | BeautifulSoup로 HTML → 텍스트 추출, 기존 인덱싱 파이프라인 재사용 | NotebookLM 동등 |
| **플래시카드 모드** | Pipeline F의 프롬프트만 변경 ("앞면: 질문 / 뒷면: 답" 형식) | 학습 도우미 강화 |
| **답변 내보내기** | 마크다운/텍스트 파일로 답변 저장 버튼 | 실용성 |
| **CAD α 슬라이더 UI** | 프론트엔드에 슬라이더 추가, API로 값 전달 | 시연 임팩트 큼 |

### 보통 (반나절~1일)

| 기능 | 어떻게 | 효과 |
|------|--------|------|
| **YouTube 자막 입력** | youtube-transcript-api → 자막 텍스트 추출 → 인덱싱 | 강의 동영상 지원 |
| **PPT 자동 생성** | python-pptx + Pipeline E 요약 결과를 슬라이드로 변환 | 발표 준비 도우미 |
| **키워드 추출 + 워드클라우드** | TF-IDF/YAKE로 핵심 키워드 → 시각화 | 논문 개요 파악 |
| **비교 표 시각화** | Pipeline C 결과를 정형 표로 렌더링 (지금은 텍스트) | 비교 분석 강화 |
| **다크모드** | CSS 변수 이미 있음, 토글 버튼만 추가 | UX |

### 어려움 (2~3일)

| 기능 | 어떻게 | 효과 |
|------|--------|------|
| **음성 입력 (STT)** | Web Speech API (브라우저 내장) → 텍스트 변환 | 접근성 |
| **TTS 답변 읽기** | Web Speech API 또는 외부 TTS | 오디오 오버뷰 라이트 버전 |
| **PDF 내 형광펜** | pdf-lib 기반 annotation | PDF 뷰어 강화 |
| **인용 그래프 시각화** | D3.js/vis.js로 인용 관계 네트워크 그래프 | 시각적 임팩트 |

### 안 하는 게 나은 것

| 기능 | 이유 |
|------|------|
| 오디오 팟캐스트 생성 | TTS 모델 비용 + 품질 보장 어려움. NotebookLM의 핵심이지만 우리 차별점 아님 |
| Google Drive 연동 | OAuth 복잡 + 유지보수 부담. 파일 업로드로 충분 |
| 실시간 협업 | WebSocket + CRDT 필요. 완전 다른 프로젝트 |
| 스캔 PDF OCR | PaddleOCR/Tesseract 통합은 가능하지만 졸업작품 범위 초과 |

## M-RAG만 되고 NotebookLM이 안 되는 것

이건 우리 차별점이니까 시연 때 강조할 것:

| 기능 | 왜 중요한지 |
|------|-----------|
| **라우트 배지** | "이 답변이 어떤 전략으로 나왔는지" 보여줌. 연구 도구로서 신뢰성 |
| **CAD 환각 억제** | LLM이 지어내는 걸 구조적으로 막음. 학술 도메인에서 핵심 |
| **SCD 언어 이탈 방지** | 영어 논문 + 한국어 질문 상황에서 답변이 영어로 빠지는 걸 방지 |
| **섹션 필터 검색** | "Method 섹션에서만 찾아줘" 가능. NotebookLM은 전체 검색만 |
| **arXiv 자동 인덱싱** | 인용 논문을 자동으로 다운로드해서 검색 가능하게 만듦 |
| **특허 전용 기능** | 청구항 인식, 인용 특허 파싱, 유사 특허 검색 |
| **퀴즈 CAD 강제** | 문제를 만들 때도 환각 억제 → 원문에 있는 내용으로만 출제 |
| **라우트별 추천 질문** | 파이프라인 맥락에 맞는 후속 질문 제안 (퀴즈 후 → 해설 / 인용 후 → 핵심 기여) |
| **완전 오픈소스** | 전체 코드 공개. NotebookLM은 AI 구조 미공개 |
| **로컬/프라이빗 배포** | 민감한 자료도 서버 밖으로 안 나감 |
| **평가 프레임워크 내장** | RAGAS + Ablation으로 답변 품질 정량 평가 가능 |

| 기능 | **M-RAG** | NotebookLM | SciRAG | PaperQA2 | CG-RAG |

| 인용 추적 | arXiv 자동 인덱싱 (Pipeline D) | 없음 | 인용 인식 합성 | 없음 | 다중 홉 (영어) |
| 환각 억제 | CAD (적응적 α) | 미공개 | 없음 | 신뢰도 점수 | 없음 |

| 한국어 지원 | 완전 (MIDM + BGE-M3) | 부분 | 없음 | 없음 | 없음 |
| 동적 파이프라인 | 6경로 (A~F) | 불명 | 단일 | 단일 | 단일 |
| 퀴즈/플래시카드 | CAD 강제 + 플래시카드 | 플래시카드 | 없음 | 없음 | 없음 |

```
M_RAG/
├── frontend/           React + TypeScript + Vite (3패널 UI)
│   └── src/
│       ├── components/
│       │   ├── source/    소스 패널 (업로드, 문서 목록)

│       │   ├── history/   대화 기록
│       │   └── layout/    전체 레이아웃
│       ├── stores/        Zustand 상태관리
│       ├── api/           API 클라이언트
│       └── i18n/          한/영 번역
├── backend/
│   ├── modules/        13개 RAG 모듈
│   ├── pipelines/      6개 파이프라인 (A~F)
│   ├── api/            FastAPI 서버
│   ├── evaluation/     RAGAS + Ablation 평가
│   └── scripts/        실험/배포 스크립트
├── docker-compose.yml  원클릭 배포
└── docs/               문서
```

## 시연 시나리오 (5분 데모)

1. **논문 업로드** → 섹션 태그 자동 표시 확인
2. **"이 논문의 핵심 기여가 뭐야?"** → Route A 배지 + 답변 + 추천 질문 3개
3. **추천 질문 클릭** → 자동 전송 → 새 답변
4. **"실험 결과를 보여줘"** → Route B (섹션 특화) + PDF 하이라이트
5. **"이 내용으로 5문제 만들어줘"** → Route F (퀴즈) + 객관식 5문제
6. **"참고문헌 분석해줘"** → Route D + 참고문헌 탭에 인용 논문 목록
7. **CAD 효과 설명** → "환각 억제가 켜져 있어서 검색된 내용에만 기반합니다"
