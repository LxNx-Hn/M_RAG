# M-RAG 개념 정리

- 문서 기준 2026-04-25
- 저장소에서 실제로 쓰는 개념만 추려 정리

## RAG

- 검색으로 관련 문맥을 찾고 그 문맥으로 답변을 생성하는 방식
- 이 저장소에서는 질문마다 다른 파이프라인으로 검색과 생성을 조합

## 임베딩

- 텍스트를 벡터로 바꿔 의미 기반 유사도 검색에 쓰는 표현
- 이 저장소에서는 `BAAI/bge-m3` 사용

## BM25

- 단어 일치 중심의 전통적 검색 점수
- Dense 검색이 놓치는 정확한 키워드를 보완

## RRF

- Dense 순위와 BM25 순위를 결합하는 융합 방식
- `HybridRetriever` 에서 사용

## 리랭킹

- 1차 검색 후보를 다시 점수화해 상위 문맥만 남기는 단계
- `cross-encoder/ms-marco-MiniLM-L-6-v2` 사용

## 라우팅

- 질문 의도를 보고 어떤 파이프라인을 탈지 결정하는 단계
- A 일반 QA
- B 섹션 QA
- C 비교
- D 인용 추적
- E 전체 요약
- F 퀴즈 생성

## 컨텍스트 압축

- 검색 결과를 LLM 입력 길이에 맞게 줄이는 단계
- 압축과 정렬을 거쳐 중요한 문맥을 앞쪽에 배치

## CAD

- 컨텍스트가 없는 출력 경향을 억제해 환각을 줄이는 디코딩 보정
- `cad_decoder.py` 에 구현

## SCD

- 목표 언어가 아닌 토큰을 약하게 억제해 언어 이탈을 줄이는 보정
- `scd_decoder.py` 에 구현

## SSE 스트리밍

- 답변 생성 중 토큰을 순차적으로 전달하는 방식
- 프론트 채팅 패널에서 실시간 렌더링

## Master Runner

- 로컬 실험 전체를 순서대로 실행하는 자동 러너
- 서버 시작, 문서 인덱싱, pseudo GT 생성, Track 실행, 결과 변환, 결과 검증까지 담당
