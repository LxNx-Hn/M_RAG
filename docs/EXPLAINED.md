# M-RAG 쉽게 설명하기

- 문서 기준 2026-04-25
- 비전공자나 새 팀원이 구조를 빠르게 이해하기 위한 설명 문서

## 이 시스템은 무엇을 하나

- 논문을 올리면 그 논문을 읽고 질문에 답하는 도우미
- 단순 채팅이 아니라 논문 안에서 근거를 찾고 답하는 구조

## 왜 일반 챗봇과 다른가

- 일반 챗봇은 학습된 기억으로 답하는 비중이 큼
- M-RAG 는 먼저 문서에서 관련 부분을 찾고 그다음 답함
- 그래서 근거를 붙여 답하기 쉬움

## 왜 모듈형인가

- 질문이 항상 같은 종류가 아니기 때문
- 결과 섹션만 묻는 질문
- 두 논문을 비교하는 질문
- 참고문헌을 따라가는 질문
- 전체 요약을 만드는 질문
- 퀴즈를 만드는 질문

## 내부에서 일어나는 일

1. 문서를 작은 조각으로 나눔
2. 각 조각을 검색 가능한 형태로 저장
3. 질문을 보고 적절한 경로 선택
4. 관련 조각을 찾고 정렬
5. 필요한 문맥만 압축
6. LLM 이 최종 답변 생성

## Mini 와 Base 는 어떻게 다른가

- Mini 는 로컬 12GB급 GPU에 맞춘 기본 모델
- Base 는 더 큰 GPU에서 비교 실험이나 대형 추론에 쓰는 선택 모델
- 둘 다 유지하지만 기본 실행 경로는 Mini

## 왜 양자화를 쓰지 않나

- 현재 로컬 기준과 문서 기준을 단순하게 유지하기 위해서
- Mini 로도 연구용 검증 경로를 운영할 수 있기 때문
